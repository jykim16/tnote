# tnote

Per-tmux-window notepad. Each window gets its own persistent markdown note, opened in a popup anchored to the top-right corner. Press the same key to close it.

```
┌─────────────────────────────────────────────────┐
│  work: api-server          tnote: api-server    │
│                           ┌───────────────────┐ │
│  $ cargo run              │ ## TODO           │ │
│  Compiling...             │ - add auth header │ │
│                           │ - check rate lim  │ │
│                           │                   │ │
└───────────────────────────┴───────────────────┘ │
```

## Requirements

- tmux 3.2+
- Rust toolchain (to build)

## Install

**One-liner (macOS/Linux):**

```sh
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/jkim/tnote/releases/latest/download/tnote-installer.sh | sh
```

**From source:**

```sh
git clone https://github.com/jkim/tnote
cd tnote
cargo install --path .
```

- `cargo build --release` — compiles the binary into `target/release/tnote` but does not install it. Useful for testing before installing.
- `cargo install --path .` — compiles and copies the binary to `~/.cargo/bin/tnote`, making it available on your `PATH`. Run this again after any code change to update the installed binary.
- `make install` — runs `cargo build --release` then copies `target/release/tnote` to `~/bin/tnote`. Use this instead of `cargo install` if you prefer the binary in `~/bin/` rather than `~/.cargo/bin/`.

**After installing the binary**, wire up the tmux keybinding:

```sh
tnote install
```

To make Ctrl+N persist across tmux restarts, add to `~/.tmux.conf`:

```
bind-key -n C-n run-shell ~/.tnotes/toggle.sh
```

## Usage

```
tnote                  Open/toggle popup for the current window
tnote name <name>      Name this window's note (also renames the tmux window)
tnote show             Print note contents inline
tnote clear            Clear this window's note
tnote list             List all notes with line counts
tnote path             Print the note file path
tnote install          Write toggle.sh and bind Ctrl+N in tmux
tnote uninstall        Remove toggle.sh and unbind Ctrl+N in tmux
tnote help             Show help
```

## How it works

**Window keys** — each note is keyed to `<session>-<window_index>` (e.g. `work-0`, `work-1`), retrieved from tmux at runtime. Notes survive across shell sessions as long as the tmux window index stays the same.

**Named notes** — `tnote name <n>` assigns a stable name to the current window's note. This writes a `.link` file mapping the window key to the name, stores the note as `named-<n>.md`, and renames the tmux window. Two windows can share a note by naming them the same thing.

**Toggle** — running `tnote` (or pressing Ctrl+N after `tnote install`) is a toggle. If the popup is already open, it sends the appropriate save+quit keystrokes to the editor and closes the popup. If it's closed, it opens it.

**Outside tmux** — if `$TMUX` is not set, `tnote` falls back to opening the editor inline in the current terminal instead of spawning a popup.

## File layout

```
~/.tnotes/
  work-0.md              unnamed note for window 0 of session "work"
  work-1.md              unnamed note for window 1
  named-api-server.md    a named note
  work-1.link            contains "api-server" — maps window 1 to that named note
  toggle.sh              generated by 'tnote install'
```

Notes are plain markdown files. You can read, edit, grep, or back them up with any standard tool.

## Configuration

| Variable | Default | Description |
|---|---|---|
| `TNOTE_DIR` | `~/.tnotes` | Directory where notes are stored |
| `TNOTE_EDITOR` | `vim` | Editor to open inside the popup |
| `TNOTE_WIDTH` | `62` | Popup width in columns |
| `TNOTE_HEIGHT` | `22` | Popup height in lines |

Set these in your shell profile or in `~/.tmux.conf` before the `run-shell` line:

```sh
export TNOTE_EDITOR=nano
export TNOTE_WIDTH=80
```

## Supported editors

Save+quit is sent automatically when toggling closed:

| Editor | Keys sent |
|---|---|
| vim, nvim, vi | `Escape` `:wq` `Enter` |
| nano | `Ctrl+O` `Enter` `Ctrl+X` |
| micro | `Ctrl+S` `Ctrl+Q` |
| helix (hx) | `Escape` `:wq` `Enter` |
| anything else | popup is killed immediately |
